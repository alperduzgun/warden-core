## 7. Auto-Improve Akışı

### 7.1 Rules Autoimprove (#648)

**Komut:** `warden rules autoimprove`

**Akış:**
1. `.warden/corpus/` içindeki düşük güvenilirlikli bulguları tarar
2. LLM FP bastırma pattern'i önerir
3. Önerilen pattern'i tam corpus'a karşı doğrular
4. **Keep-or-revert:** F1 düşmemeli; düşerse geri alır
5. Onaylanan pattern'leri `fp_exclusions.py`'e yazar

**Tetikleme:** Manuel (`warden rules autoimprove`) veya scan üzerinde `--auto-improve` flag'i.

### 7.2 Resilience Frame Autoimprove (#657)

**ResilienceFrame:** `src/warden/validation/frames/resilience/resilience_frame.py`
- Static check'ler LLM'den **önce** çalıştırılır (performans + determinizm)
- #657 ile autoimprove desteği eklendi
- Security frame ile aynı keep-or-revert döngüsü

### 7.3 `--report-fp` Kullanıcı Akışı

```
kullanıcı çalıştırır: warden scan . --report-fp security-sql-injection-3

1. Bulgu mevcut scan'de aranır (fallback: .warden/cache/)
2. Kod parçacığı .warden/corpus/<proje>_reported_fp.py'ye yazılır
   corpus_labels: {check_id: 0} ile
3. Autoimprove döngüsü .warden/corpus/'a karşı çalıştırılır
4. Pattern doğrulanır → fp_exclusions.py'e yazılır
5. Sonraki scan: pattern katmanında bastırılır
```

## 15. Hatalar ve Edge Case'ler

### 15.1 Provider Rate Limit

`global_rate_limiter.py` + LLM registry'deki circuit breaker ile yönetilir. Birincil rate-limited olursa sonraki provider'a düşer.

### 15.2 Corpus Boş

`CorpusResult.overall_f1` metrik toplanmadıysa `0.0` döndürür (`runner.py:91-94`). CI gate `--min-f1` başarısız olur.

### 15.3 LLM Hata Cevabı

Frame runner `@async_error_handler` decorator'ü ile istisnaları yakalar (`frame_runner.py`, `shared.infrastructure.error_handler`'dan import). Pipeline'i çökertmek yerine kısmi sonuçlar döndürür.

### 15.4 Cross-File Truncation (Son Düzeltme)

**Commit:** `f204c77` — "fix(orphan): fix cross-file corpus truncated to 5-file chunk, causing mass FPs"

OrphanFrame'in cross-file analizi corpus'u 5-dosya chunk'larıyla sınırlıyordu, çok dosyalı projelerde mass FP oluşturuyordu. Chunk limiti kaldırılarak/artırılarak düzeltildi.

### 15.5 Path Traversal Saldırısı

Birden fazla katmanda korunur:
- **PromptManager:** `templates_dir.resolve()` + doğrulama (`prompt_manager.py:71`)
- **Auto-init:** YAML injection ve TOCTOU karşı sertleştirilmiş (`#534`)
- **Dosya keşfi:** `.wardenignore` ve `.gitignore`'a saygı gösterir
